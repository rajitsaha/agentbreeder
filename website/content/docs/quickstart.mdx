# Quickstart

Two paths. Pick the one that matches where you are.

| | Option A — Full local platform | Option B — Build your own agent |
|---|---|---|
| **Who it's for** | First-timers, evaluators | Engineers who want a blank slate |
| **What you get** | Full stack + 5 sample agents + dashboard | One scaffolded agent project |
| **Time** | ~3 min (first run, image pulls) | ~5 min |

---

## Prerequisites

| Requirement | Min version | Why |
|-------------|-------------|-----|
| **Python** | 3.11+ | CLI runtime |
| **Docker Desktop** | 24+ | Container orchestration (Option A requires this; Option B needs it for local deploy) |

```bash
python3 --version   # need 3.11+
docker info         # must succeed — if it errors, open Docker Desktop first
```

<Callout type="warn" title="Docker not running?">
  Open the **Docker Desktop** app before running any `docker` commands. On Linux: `sudo systemctl start docker`. The `agentbreeder quickstart` command detects Docker and guides you through installing it if it is not found.
</Callout>

## 1. Install

Pick the method that matches your environment:

| Method | Command |
|--------|---------|
| **pip** (Python 3.11+) | `pip3 install agentbreeder` |
| **Homebrew** (macOS) | `brew tap agentbreeder/agentbreeder && brew install agentbreeder` |
| **Docker** | `docker run rajits/agentbreeder-cli --help` |
| **npm** | `npm install @agentbreeder/sdk` |

> **`agentbreeder: command not found` after pip install?** pip's script directory may not be in your PATH. Run:
> ```bash
> python3 -c "import sysconfig; print(sysconfig.get_path('scripts'))"
> ```
> Add that path to your `PATH` (e.g. in `~/.zshrc`), then open a new terminal.
> On macOS, **Homebrew is the easiest install** — it handles PATH automatically.

All install options → [How-To: Install →](how-to#install-agentbreeder)

## Option A — Full Local Platform

One command bootstraps the entire local stack:

```bash
agentbreeder quickstart
```

That single command:

1. Detects Docker / Podman — shows OS-specific install instructions if neither is found
2. Checks for Ollama or cloud API keys — suggests `agentbreeder setup` if neither found
3. Creates `.env` with secure random secrets if not present
4. Starts the full stack: **API · Dashboard · PostgreSQL · Redis · ChromaDB (RAG) · Neo4j (GraphRAG) · MCP servers · LiteLLM gateway**
5. Waits for all services with a live progress bar
6. Seeds a ChromaDB knowledge base and a Neo4j knowledge graph with sample data
7. Deploys 5 sample agents: `assistant`, `rag-agent`, `graph-agent`, `search-agent`, `a2a-orchestrator`
8. Opens the visual dashboard at `http://localhost:3001`

Takes ~3 minutes on first run (image pulls). After that:

```bash
agentbreeder chat assistant                   # plain conversation
agentbreeder chat rag-agent                   # ask questions about AgentBreeder docs
agentbreeder chat graph-agent                 # query the knowledge graph
agentbreeder chat a2a-orchestrator            # let the orchestrator route your question
agentbreeder chat my-agent --local            # chat via Ollama — no API server needed
```

Deploy to cloud from the same local setup:

```bash
agentbreeder quickstart --cloud aws           # local + deploy to AWS ECS Fargate
agentbreeder quickstart --cloud gcp           # local + deploy to GCP Cloud Run
agentbreeder quickstart --cloud azure         # local + deploy to Azure Container Apps
```

<Callout type="info" title="No API key? Use Ollama.">
  `agentbreeder quickstart` works without any cloud credentials. Install [Ollama](https://ollama.com),
  run `ollama pull llama3.2`, and the stack will use it automatically via the LiteLLM gateway.
  Run `agentbreeder setup` first if you want a guided Ollama + provider key wizard.
</Callout>

## Option B — Build Your Own Agent

For engineers who want a blank-slate agent project:

### 1. Configure Providers

```bash
agentbreeder setup
```

Interactive wizard that guides you through Ollama installation (with model picker) and cloud API key configuration (with live key validation).

### 2. Scaffold a New Agent

```bash
agentbreeder init
```

The wizard asks 5 questions:

| # | Question | Example |
|---|----------|---------|
| 1 | Agent name | `support-agent` |
| 2 | Framework | LangGraph |
| 3 | Cloud target | Local |
| 4 | Team | `engineering` |
| 5 | Owner email | `alice@company.com` |

Generated project:

```
support-agent/
├── agent.yaml          # The only file AgentBreeder needs
├── agent.py            # Working agent code for your framework
├── requirements.txt
├── .env.example
└── README.md
```

### 3. Validate

```bash
agentbreeder validate agent.yaml
```

Runs three checks: YAML syntax → JSON Schema → semantic validation (framework supported, team exists).

### 4. Deploy

```bash
agentbreeder deploy agent.yaml --target local     # Deploy locally with Docker
agentbreeder deploy agent.yaml --target aws       # AWS ECS Fargate
agentbreeder deploy agent.yaml --target gcp       # GCP Cloud Run
agentbreeder deploy agent.yaml --target azure     # Azure Container Apps
```

The 8-step atomic pipeline:

```
✅  YAML parsed & validated
✅  RBAC check passed
✅  Dependencies resolved
✅  Container built (langgraph runtime)
✅  Deployed to Docker Compose
✅  Health check passed
✅  Registered in org registry
✅  Endpoint returned
```

If any step fails, the entire deploy rolls back.

### 5. Chat

```bash
agentbreeder status                              # see your agent's endpoint and health
agentbreeder logs support-agent --follow         # tail live logs
agentbreeder chat support-agent                  # interactive chat (requires API server)
agentbreeder chat support-agent --local          # chat via Ollama — no server needed
```
